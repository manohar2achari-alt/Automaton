Overview

Azure App Registration client secrets expire periodically. Unmanaged expirations can cause outages for apps and services.

This automation detects expiring secrets at 60, 30, and 7 days before expiry, automatically rotates secrets, and sends notifications to admins.

How It Works

  • Automation Account runbook runs daily
  • Uses Managed Identity with Graph API permissions
  • Detects secrets expiring in 60 / 30 / 7 days
  • Automatically creates new secret if needed
  • Sends notifications to administrators (Logic App / Teams / Email)

[Automation Account MI] ---reads---> [App Registrations]
                                \
                                 ---> renews secrets
                                  \
                                   ---> sends email alert

Step-by-Step Guide

1. Create Azure Automation Account

Create a PowerShell 7 Automation Account with System-assigned Managed Identity.

2. Assign Managed Identity Permissions

Grant Graph API roles:

  • Application.ReadWrite.All
  • Directory.Read.All

3. Configure Notification Webhook

Provide a Logic App HTTP trigger, Teams, or SendGrid endpoint to receive notifications.

4. Schedule Runbook

Runbook should execute daily and check secrets against 60, 30, and 7 day thresholds.
